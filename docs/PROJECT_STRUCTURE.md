# Project Structure

This document describes the cleaned and organized folder structure of the project.

## Directory Structure

```
/
├── docs/                    # All documentation files (.md)
│   ├── README.md
│   ├── AUDIT_SUMMARY.md
│   ├── AUDIT_REPORT.md
│   ├── DEPLOYMENT_GUIDE.md
│   ├── SUPABASE_SCHEMA.md
│   ├── Guidelines.md
│   ├── Attributions.md
│   └── PROJECT_STRUCTURE.md (this file)
│
├── scripts/                 # Deployment and test scripts
│   └── smoke-tests.sh
│
├── src/                     # Frontend source code
│   ├── components/          # React components
│   │   ├── ui/              # UI component library
│   │   └── figma/           # Figma-related components
│   ├── utils/               # Utility functions
│   │   └── supabase/        # Supabase client config (frontend)
│   ├── styles/              # Global styles
│   ├── App.tsx              # Main app component
│   ├── main.tsx             # Entry point
│   └── index.css            # Main CSS file
│
├── backend/                 # Backend/server code
│   └── supabase-functions/  # Supabase Edge Functions
│       └── server/          # Server functions
│           ├── index.tsx
│           └── kv_store.tsx
│
├── build/                   # Build output (generated, do not edit)
├── node_modules/            # Dependencies (generated, do not edit)
│
├── index.html               # HTML entry point
├── package.json             # Node.js dependencies and scripts
├── package-lock.json        # Locked dependency versions
├── vite.config.ts           # Vite configuration (must be at root)
├── vercel.json              # Vercel deployment config (must be at root)
├── .gitignore              # Git ignore rules
└── .env.example            # Environment variables template (if exists)
```

## Key Decisions

### Files at Root Level
- **vite.config.ts**: Must remain at root as Vite expects it there
- **vercel.json**: Must remain at root as Vercel expects it there
- **package.json**: Standard location for Node.js projects
- **index.html**: Entry point for Vite
- **.gitignore**: Standard location

### Documentation (`/docs/`)
All markdown documentation files have been moved here for better organization:
- Project documentation
- Audit reports
- Deployment guides
- Schema documentation
- Guidelines and attributions

### Backend (`/backend/`)
Backend/server code has been separated from frontend:
- Supabase Edge Functions are now in `/backend/supabase-functions/`
- This makes it clear what code runs on the server vs client

### Frontend (`/src/`)
Frontend code remains in `/src/`:
- Components, utilities, and styles stay here
- Supabase client configuration remains in `src/utils/supabase/` as it's used by the frontend

### Build Artifacts
- `/build/`: Generated by `vite build`, do not edit
- `/node_modules/`: Generated by `npm install`, do not edit

## Import Paths

### Frontend Imports
- Components: `import { Component } from './components/Component'`
- Utils: `import { util } from './utils/util'`
- Supabase config: `import { projectId } from './utils/supabase/info'`

### Backend Imports
- Backend functions are independent and don't need to import from frontend

## Development Workflow

1. **Frontend Development**: Work in `/src/`
2. **Backend Development**: Work in `/backend/`
3. **Documentation**: Update files in `/docs/`
4. **Scripts**: Add deployment/test scripts to `/scripts/`

## Build Process

The build process remains unchanged:
- `npm run dev`: Starts development server
- `npm run build`: Builds to `/build/` directory
- Configuration files at root are automatically found by build tools
